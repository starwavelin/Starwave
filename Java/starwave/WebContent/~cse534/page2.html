What's New?

Mar 24, 2015

Flash with Sandbox

  • We compiled the Flash source into an AIR application this time. If a PC user installs this AIR app, he is able to realize HTTP URLRequest locally on his machine.
  • Then, we further found a way to compile the Flash source directly to .exe file, which a PC user can directly execute it to realize HTTP URLRequest, without installing any feature.
  • Finally, we made our Flash plug-in an in-browser AIR application, which can be viewed here

Mar 18, 2015

  • To solve the problem we discovered on Mar 12, we investigated the cross domain policy for some other similar techniques, including HTML5, PHP, and JavaScript.
  • We then figured out the possible solution to this problem would be: Flash with Sandbox, Java Applet, or Google Chrome Extension.

Mar 12, 2015

  • We upload the swf file of the Adobe Flash App with HTTP implemented onto our server, you can view it here.
  • But, it has the limitation of retrieving other sites except the site hosting it, due to the cross domain security policy of Flash.
